1. Project Overview

CineNova is a full-stack web application that serves as a unified streaming platform aggregator. It allows users to connect their streaming subscriptions (Netflix, Disney+, HBO Max, Amazon Prime, Hulu, Apple TV+) and browse content from all platforms in a single, unified interface.

Key Features

  • 🔐 User Authentication — Secure signup/login with bcrypt password hashing
  • 🔗 Platform Connection — Connect and manage multiple streaming subscriptions
  • 🔍 Universal Search — Search movies/shows across all connected platforms
  • 📋 Watchlist Management — Add/remove movies to a persistent watchlist stored in MongoDB
  • 🎬 Real Movie Data — Integration with OMDb API for live movie posters, ratings, and descriptions
  • 📱 Responsive Design — Works seamlessly across desktop, tablet, and mobile
  • ⚙️ Account Management — Settings modal with sign-out and account deletion

4. Database Design

The application uses MongoDB Atlas (cloud) as the database, accessed through the Mongoose ODM library.

User Schema

const UserSchema = new mongoose.Schema(
  {
    name: {
      type: String,
      required: [true, 'Please provide a name'],
    },
    email: {
      type: String,
      required: [true, 'Please provide an email'],
      unique: true,
    },
    password: {
      type: String,
      required: [true, 'Please provide a password'],
    },
    connectedPlatforms: {
      type: [String],   // e.g. ["netflix", "disney", "hbo"]
      default: [],
    },
    watchlist: {
      type: [String],   // Array of IMDB IDs
      default: [],
    },
  },
  { timestamps: true }  // Adds createdAt & updatedAt
);
📝 Note The connectedPlatforms field stores platform IDs as strings (e.g., "netflix", "disney"), while watchlist stores IMDB IDs (e.g., "tt1234567") to reference movies across platforms.

Database Connection — Singleton Pattern

To prevent connection leaks during development hot-reloads, the connection is cached globally:

let cached = (global as any).mongoose;

async function connectToDatabase() {
  if (cached.conn) return cached.conn;  // Reuse existing

  if (!cached.promise) {
    cached.promise = mongoose.connect(MONGODB_URI, {
      bufferCommands: false,
      serverSelectionTimeoutMS: 10000,
    }).then((mongoose) => mongoose);
  }

  cached.conn = await cached.promise;
  return cached.conn;
}

Key Code — AnimatedCounter with IntersectionObserver

function AnimatedCounter({ target, suffix }) {
  const [count, setCount] = useState(0);
  const ref = useRef(null);
  const [started, setStarted] = useState(false);

  useEffect(() => {
    const observer = new IntersectionObserver(
      ([entry]) => {
        if (entry.isIntersecting && !started)
          setStarted(true);
      },
      { threshold: 0.3 }
    );
    if (ref.current) observer.observe(ref.current);
    return () => observer.disconnect();
  }, []);

  useEffect(() => {
    if (!started) return;
    let current = 0;
    const increment = target / 60;
    const timer = setInterval(() => {
      current += increment;
      if (current >= target) {
        setCount(target); clearInterval(timer);
      } else setCount(Math.floor(current));
    }, 25);
  }, [started, target]);

  return <div ref={ref}>{count}{suffix}</div>;
}

The signup page collects the user's name, email, password, and password confirmation. It validates that the passwords match before sending the request.

Backend — Signup API Route (/api/auth/signup)

export async function POST(request: Request) {
  const { name, email, password } = await request.json();
  await connectToDatabase();

  // Check if user already exists
  const existingUser = await User.findOne({ email });
  if (existingUser) {
    return NextResponse.json(
      { message: 'User already exists' },
      { status: 409 }
    );
  }

  // Hash password with bcrypt (salt rounds = 10)
  const salt = await bcrypt.genSalt(10);
  const hashedPassword = await bcrypt.hash(password, salt);

  // Create user document in MongoDB
  const newUser = await User.create({
    name, email, password: hashedPassword
  });

  // Set HTTP-only session cookie
  response.cookies.set('userId', newUser._id.toString(), {
    httpOnly: true,            // Not accessible via JS
    secure: process.env.NODE_ENV === 'production',
    maxAge: 60 * 60 * 24 * 7,  // 1 week
    path: '/',
  });
}
⚠️ Important — Security Passwords are never stored in plain text. They are hashed using bcrypt with a salt factor of 10 before saving to the database. The session cookie is httpOnly, preventing XSS attacks from accessing it.

7. User Authentication — Login

Fig 7.1 — Login page with email and password fields, "Forgot Password" link, and redirect to signup

The login page authenticates returning users by verifying their email and password against the database. If a user is not found (404), they are automatically redirected to the signup page.

Backend — Login API Route (/api/auth/login)

export async function POST(request: Request) {
  const { email, password } = await request.json();
  await connectToDatabase();

  // Find user by email
  const user = await User.findOne({ email });
  if (!user) {
    return NextResponse.json(
      { message: 'User not found' }, { status: 404 }
    );
  }

  // Compare submitted password with stored hash
  const isMatch = await bcrypt.compare(password, user.password);
  if (!isMatch) {
    return NextResponse.json(
      { message: 'Invalid credentials' }, { status: 401 }
    );
  }

  // Set session cookie and return success
  response.cookies.set('userId', user._id.toString(), {
    httpOnly: true,
    maxAge: 60 * 60 * 24 * 7,
    path: '/',
  });
}

function OnboardingScreen({ onComplete }) {
  const [selected, setSelected] = useState(new Set());

  const handleContinue = async () => {
    const ids = Array.from(selected);

      method: "POST",

All API routes follow RESTful conventions and use cookie-based authentication. Protected routes read the userId cookie to identify the logged-in user.

Watchlist API — MongoDB Atomic Operations

Uses MongoDB's $addToSet (prevents duplicates) and $pull (removes items) operators for data integrity:

// ADD to watchlist — prevents duplicate entries
const user = await User.findByIdAndUpdate(
  userId,
  { $addToSet: { watchlist: movieId } },
  { new: true }
).select('-password');  // Exclude password from response

// REMOVE from watchlist
const user = await User.findByIdAndUpdate(
  userId,
  { $pull: { watchlist: movieId } },
  { new: true }
).select('-password');

10. External API Integration (OMDb)

The application integrates with the OMDb (Open Movie Database) API to fetch real movie data including posters, ratings, genres, and descriptions.

Movie Data Interface

export interface Movie {
  id: string;           // IMDB ID (e.g., "tt1234567")
  title: string;
  poster_path: string;  // URL to movie poster image
  description: string;  // Plot summary
  year: string;
  rating: string;       // IMDB rating
  genre: string;        // Comma-separated genres
  runtime: string;
  match: number;        // Match percentage (85-100%)
  trailerUrl: string;   // YouTube search URL
}

Fetching Movie Data from OMDb

const fetchMovies = async (list: string[]) => {
  const results = await Promise.all(
    list.map(async (title) => {
      const res = await fetch(
        `https://www.omdbapi.com/?t=${title}&apikey=${API_KEY}`
      );
      const data = await res.json();
      if (data.Response === "True") {
        return {
          id: data.imdbID,
          title: data.Title,
          poster_path: data.Poster,
          description: data.Plot,
          rating: data.imdbRating,
          genre: data.Genre,
          // ... other fields
        };
      }
      return null;
    })
  );
  return results.filter((m) => m !== null);
};
